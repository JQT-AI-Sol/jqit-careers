# デザインシステム — JQIT 採用サイト（v2: エディトリアル・ミニマル）

- **対象**: 第1弾（Phase 4 / UI/UX設計）
- **方針（確定）**: ビズリーチ採用サイト級の**清潔感・上質感・スタイリッシュさ**を主役にする。**明朝体ベースのエディトリアル・ミニマル**。ほぼモノクロで構成し、JQITレッドは「ここぞ」の精密な差し色に限定。
- **採用モック**: `wireframes/top-mockup-v2.html`（ユーザー承認済み）。v1（`top-mockup.html`／太ゴシック・赤×黒の力強い系）は不採用。
- **ブランド根拠**: ロゴ抽出 JQITレッド `#E60012`。

---

## 1. デザイン原則

1. **余白を恐れない** — セクション縦padding 120–150px。要素間もゆったり。空白が上質感を生む。
2. **明朝で語る** — 見出し・キーステートメントは明朝（Noto Serif JP）。軽〜中ウェイト。
3. **色は引き算** — 白70%以上／ニアブラック文字／赤は数%。赤は logo dot・主要CTA・極細アクセント線・強調語のみ。
4. **線で仕切る、箱で囲わない** — 影付きカードを避け、極細罫線（1px）と余白で構造を作る。
5. **静かなモーション** — fade/slide はごく控えめ。`prefers-reduced-motion` を尊重。

**コントラスト**: 本文 AA 以上。

## 4. レイアウト / スペーシング

- スケール（px）: `4, 8, 12, 16, 24, 32, 48, 72, 96, 120, 150`
- コンテナ最大幅: `1160px`（左右ガター: PC 32px / SP 22px）
- セクション縦padding: PC `120–150px` / SP `80px`
- セクション見出しブロック（kicker+H2+lead）下の余白: `72px`
- グリッド: 強み=4カラム（罫線区切り）、魅力=3カラム、職種=リスト行、社員の声=3カラム。

---

## 5. 角丸・線・影・モーション

| 項目 | 値 |
|------|----|
| 角丸 | `2px`（ほぼ直角＝シャープで上質）。画像も2px |
| 罫線 | `1px solid var(--c-line)`。区切りの主役 |
| 影 | **原則なし**。要素は罫線と余白で分離 |
| モーション | スクロールイン fade+up 8–16px / 0.6s ease。ホバー 0.25s（リンクは下線が赤に・矢印が右へ少し動く） |
| フォーカス | `outline:2px solid var(--c-red); outline-offset:2px`（A11y必須・F-017） |

## 6. コンポーネント仕様（v2）

### Header / GlobalNav（S-002）
- 白（半透明＋blur）・下に極細罫線・sticky。左ロゴ（赤dot＋JQIT/Archivo）、右にサンセリフのナビ＋赤の小さめ「エントリー」ボタン。
- スマホ: ハンバーガー → 白基調のフルスクリーンメニュー（明朝リンク）。

### Hero（S-001）
- **白基調**。kicker（赤線＋英字）→ 明朝の大見出し（赤は強調語1語のみ）→ サンセリフのサブ → CTA（赤ボタン＋テキストリンク「職種を見る →」）。
- その下にワイドな**モノクロ写真**（grayscale）。左下に赤の縦ライン＋英字キャプション。

### Button（S-021）
- **Primary**: 背景 `--c-red` / 白文字 / 角丸2px / サンセリフ太字。主要CTAのみ。
- **Outline**: 黒枠 / 黒文字 / ホバーで黒背景・白文字。
- **テキストリンク（arrow）**: 黒文字＋下線、ホバーで赤＋矢印が右へ。多用はこちらで軽く。

### 強み（S-003 / 罫線コラム）
- 箱なし。上下に罫線、4カラムを縦罫で仕切らず余白で分離。各: 赤の番号（01..）＋明朝H3＋サンセリフ小文。

### 職種リスト（S-006/007 / エディトリアル行）
- **リスト行**: `番号 | 本文（英字カテゴリ＝赤小・明朝職種名・サンセリフ説明・「—未経験OK / —経験者歓迎」）| 矢印`。行間は罫線で区切り、ホバーで背景白＋左inset＋矢印が赤で右移動。

### 社員の声（S-008）
- 3カラム。上にモノクロ写真（左下に赤4pxライン）→ 英字role（赤小）→ 明朝の見出し（「』調のコピー）→ サンセリフ本文。

### 数字（Culture / S-009）
- 明朝の大きな数値（単位は赤・小）＋サンセリフの小ラベル。写真はモノクロのモザイクグリッド。

### CTA（S-013 導線）
- **淡色背景**（`--c-bg-soft`）・中央寄せ。kicker → 明朝の大見出し（強調語のみ赤）→ サンセリフ補足 → 赤ボタン。重い黒ブロックにしない。

### FormField（S-012）
- ラベル上・サンセリフ。入力は1px罫線・2px角丸・フォーカスで赤枠。エラーは赤。必須「*」。

### FAQ（S-011）
- 罫線区切りのアコーディオン。＋/−、開時に左に赤の細ライン。

### Footer（S-020）
- `--c-ink` 背景・控えめ。ロゴ＋明朝のタグライン、Archivoの小見出し（Recruit/Company/Contact）、極細上罫線のコピーライト行。

## 7. 画像・ビジュアル方針

- 写真は原則 **grayscale（モノクロ）** に統一し、赤を差し色に。エディトリアルで上質に。
- 装飾は赤の極細ライン・小さな幾何のみ。過度なグラフィックは置かない。
- アイコンは最小限（線画）。番号（01..）で構造を示す方を優先。

---

## 8. アクセシビリティ（F-017）

- コントラストAA、フォーカス可視化（赤outline）、キーボード操作、`alt`、ランドマーク、見出し階層、`prefers-reduced-motion`。

---

## 9. 実装への橋渡し（Phase 5）

- フォント: `next/font` で **Noto Serif JP / Noto Sans JP / Archivo** を最適化読み込み。`--font-serif` を本文/見出しの基調に。
- Tailwind `theme.extend`: 上記カラー・フォントファミリ・余白・角丸(2px)・罫線色をトークン化。本文の既定 font-family を serif に。
- 共通コンポーネントを `src/components/` に分割（Header, Hero, SectionHead, StrengthGrid, JobRow, MemberCard, Stats, FAQ, CTA, Footer, EntryForm, Button, Field）。
- モーションは控えめなユーティリティ（Intersection Observer での fade-up）。
